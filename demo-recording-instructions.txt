MemoryStudio / MemoryRoom — Demo Recording Instructions

Goal:
Produce a short demo video that proves AC1–AC4.

Recommended duration:
60–120 seconds.

Recording steps (in order):
1) Open app in Chrome (mobile preferred: Samsung S21 FE).
2) Show landing overlay and tap "Play Demo".
3) Rotate/look around to show 360 panorama.
4) Tap hotspot 1 and show card opens.
5) Tap hotspot 2 and show card opens.
6) In chat, type: "tell me about the cake".
7) Show agent text reply and TTS narration.
8) Tap "Speak" and show voice input path (or mention fallback if unsupported).
9) Toggle ambient audio on/off.
10) Tap "Auto-Author (demo)" and show endpoint/fallback response.

What to say on recording (optional script):
- "This is MemoryStudio, an offline-safe, rule-based memory room demo."
- "Hotspots open memory cards and trigger narration."
- "The agent supports typed chat and speech where available."
- "Auto-Author uses a deterministic serverless stub, no external LLM and no API keys."

Capture settings:
- Resolution: 1080p recommended.
- Frame rate: 30fps.
- Keep UI text readable.

Before recording:
- Ensure assets are present in /assets.
- Confirm browser microphone permission if testing speech input.
- Confirm device volume is audible for TTS/audio demo.

Deliverables:
- Public video link (YouTube unlisted, Loom, or equivalent).
- Add video URL to README and submission issue template.
